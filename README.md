# PRO1000
Praktisk Prosjektarbeid USN Bø - Gruppe semesteroppgave 

Prosjekttittel
JavaSaDu – En interaktiv læringsplattform for Java-programmering.

Prosjektorganisering og Roller
Prosjektleder: Henrik Premer / Ola Wold 
Produkteier: Henrik Premer / Ola Wold
Utviklere: Henrik Premer 
Kvalitets og dokumentasjonsansarlig: Ola Wold


Prosjektbakgrunn / Problemstilling
Mange IT-studenter opplever overgangen til objektorientert programmering (Java) som bratt og teoretisk tung. Eksisterende ressurser er ofte enten for omfattende (lærebøker) eller mangler progresjonsmåling knyttet til spesifikke pensumkrav.
Problemet: Studentene mangler et lavterskelsystem som kombinerer teori, interaktive tester og visuell oversikt over egen progresjon som er mer brukervennlig enn canvas. 

Prosjektmål
    • Funksjonelt system: Utvikle en web-applikasjon som lar brukere navigere gjennom ulike Java-moduler.
    • Progresjonshåndtering: Implementere lagring av brukerdata slik at studenter kan se hvilke emner de har mestret.
    • Interaktivitet: Inkludere quizer og praktiske eksempler som demonstrerer solid bruk av moderne systemutviklingsmetodikk (Scrum/Kanban).

Omfang (Scope)
Innenfor omfang:
    • Autentisering (Innlogging via GitHub eller e-post).
    • Dashboard med oversikt over progresjon.
    • Minimum tre læringsmoduler (for eksempel variabler, løkker og klasser).
    • Quiz-motor for hver modul.
    • Responsivt design for både mobil og desktop.
Utenfor omfang:
    • Ekte sanntidskompilering av Java-kode i nettleseren.
    • Sosiale funksjoner som venne-lister eller chat-funksjonalitet.

Antakelser og begrensninger 
    • Antakelser: Vi antar at studentene har grunnleggende IT-forståelse. Vi antar at Supabase dekker vårt behov for database og autentisering.
    • Begrensninger: Tidsfrist satt av semesteret. En teamstørrelse på to personer begrenser mengden innhold (moduler) vi kan produsere.

Risiko
    • Teknisk risiko: Utfordringer med integrasjon mellom Next.js og Supabase. (Tiltak: Tidlig prototyping).
    • Tidsrisiko: For mye tid brukes på innhold og tekst fremfor systemlogikk. (Tiltak: Bruke AI til generering av innhold).
    • Omfangskryp (Scope Creep): Ønske om å legge til for mange funksjoner. (Tiltak: Streng prioritering i backlogen).
